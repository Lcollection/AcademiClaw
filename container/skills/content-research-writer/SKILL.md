# Content Research Writer

Write high-quality content with research-backed citations and real-time feedback.

## When to Use

- Writing blog posts, articles, or documentation
- Creating research-backed content
- Improving existing drafts
- Developing outlines with research
- Adding citations and sources

## Workflow

### Phase 1: Outline Development

1. **Understand the goal**
   - What's the main message?
   - Who's the target audience?
   - What action should readers take?

2. **Create initial outline**
   ```markdown
   # [Title Idea]

   ## Hook
   [Compelling opening]

   ## Section 1: [Topic]
   - Key point
   - Supporting evidence needed

   ## Section 2: [Topic]
   - Key point
   - Supporting evidence needed

   ## Conclusion
   [Call to action]
   ```

3. **Research gaps**
   - Identify claims needing sources
   - Search for supporting data
   - Find expert quotes

### Phase 2: Research & Citation

```python
# For each claim that needs support:
# 1. Search for sources
# 2. Verify credibility
# 3. Extract key data
# 4. Format citation

# Citation format:
[Author/Org]([Year]). [Title]. [Source]. [URL]
```

**Credibility checklist:**
- [ ] Author has relevant credentials
- [ ] Source is peer-reviewed or reputable
- [ ] Data is recent (within 2-3 years)
- [ ] Information is corroborated elsewhere

### Phase 3: Draft Writing

**Writing principles:**

1. **Hook readers immediately**
   - Start with a surprising fact
   - Pose a provocative question
   - Tell a brief story
   - State a bold claim

2. **Maintain momentum**
   - Short paragraphs (2-4 sentences)
   - Active voice
   - Concrete examples
   - Varied sentence structure

3. **Support claims**
   - Data points with sources
   - Expert quotes
   - Case studies
   - Research findings

4. **End strong**
   - Summarize key takeaways
   - Clear call to action
   - Thought-provoking closing

### Phase 4: Iteration & Feedback

**Self-review checklist:**

```markdown
## Structure
- [ ] Hook grabs attention
- [ ] Clear progression of ideas
- [ ] Sections flow logically
- [ ] Conclusion resonates

## Content
- [ ] Claims are supported
- [ ] Citations are accurate
- [ ] Examples are relevant
- [ ] Tone is appropriate

## Style
- [ ] Sentences are clear
- [ ] No unnecessary jargon
- [ ] Active voice dominant
- [ ] Reading level appropriate

## SEO (if applicable)
- [ ] Keywords naturally integrated
- [ ] Headings are descriptive
- [ ] Meta description compelling
```

## Section Feedback Template

For each section, provide:

```markdown
### Section: [Name]

**Strengths:**
- [What works well]

**Areas for Improvement:**
- [Specific suggestions]

**Research Gaps:**
- [Claims needing sources]

**Suggestions:**
1. [Specific rewrite idea]
2. [Alternative approach]
```

## Best Practices

1. **Write first, edit later** - Get ideas down before perfecting
2. **Show, don't tell** - Use examples over abstractions
3. **One idea per paragraph** - Keep focus clear
4. **Vary sentence length** - Create rhythm
5. **End sections strong** - Leave readers wanting more

## Anti-Patterns

- ❌ Walls of text
- ❌ Unsubstantiated claims
- ❌ Jargon without explanation
- ❌ Passive voice dominance
- ❌ Weak hooks
- ❌ Abrupt endings
